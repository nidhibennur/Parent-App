# ParentApp

A parenting support web app built as part of a project at OTH Amberg-Weiden. It provides AI-powered chat, mood tracking, developmental milestone tracking, calm tools, and curated learning resources вЂ” all personalised to the parent's profile and their children's ages.

---

## Features

- **AI Chat** вЂ” Conversational parenting support powered by GPT via OpenRouter, with full chat history saved to MongoDB
- **Mood Check-in** вЂ” Daily mood logging with personalised AI advice; entries saved to your account
- **Milestone Tracker** вЂ” Developmental milestone checklist personalised to each child's age range, synced to MongoDB
- **Calm Tools** вЂ” Box breathing, reset timers, and AI-generated grounding prompts
- **Parenting Tips** вЂ” Age-filtered tips generated by AI on demand
- **Learning Hub** вЂ” Curated articles and in-app Q&A per topic
- **Emergency Page** вЂ” Grounding steps, breathing animation, and crisis resources
- **Profile** вЂ” Stores name, role, children (nickname + age range), challenges, and preferred response style

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 19, Vite, React Router v7 |
| Styling | Custom CSS |
| Backend | Python, FastAPI, Uvicorn |
| Database | MongoDB Atlas (via Motor async driver) |
| AI | OpenRouter API (GPT model) |
| Auth | JWT (python-jose) + bcrypt |

## Getting Started

### Prerequisites

- Node.js 18+
- Python 3.10+
- A MongoDB Atlas cluster (free tier works)
- An OpenRouter API key (free tier available at openrouter.ai)

---

### Backend

```bash
cd parentapp-backend

# Create and activate virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Create .env file (see Environment Variables below)
cp .env.example .env

# Start the server
uvicorn main:app --port 4000 --reload
```

The backend runs at `http://localhost:4000`.

---

### Frontend

```bash
cd parentapp-frontend

# Install dependencies
npm install

# Start the dev server
npm run dev
```

The frontend runs at `http://localhost:5173`.

---

## Environment Variables

Create a `.env` file in `parentapp-backend/` with the following:

```env
MONGODB_URI=mongodb+srv://<user>:<password>@<cluster>.mongodb.net/
OPENROUTER_API_KEY=sk-or-...
JWT_SECRET=your-long-random-secret-here
```

| Variable | Description |
|---|---|
| `MONGODB_URI` | MongoDB Atlas connection string |
| `OPENROUTER_API_KEY` | API key from openrouter.ai |
| `JWT_SECRET` | Secret used to sign JWT tokens вЂ” use a long random string in production |

## Deployment

- **Backend** вЂ” Deploy to [Render](https://render.com) as a Python web service. Set environment variables in the Render dashboard.
- **Frontend** вЂ” Deploy to [Vercel](https://vercel.com). Update the API base URL in `src/utils/` from `localhost:4000` to your Render service URL.
